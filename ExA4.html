(a) FP-growth Algorithm¶

Consider the following transaction data:

In order to find frequent itemsets with $minsup = 0.2$, we first find the support count for each item. Items with support count $\le 2$ are discarded.

{b} 7
{c} 6
{d} 5
{e} 3

The frequent items are sorted in decreasing support counts inside every transaction of the data set.

TID Items
1 {a, b}
2 {b, c, d}
3 {a, c, d, e}
4 {a, d, e}
5 {a, b, c}
6 {a, b, c, d}
7 {a}
8 {a, b, c}
9 {a, b, d}
10 {b, c, e}

Next, the algorithm scans the data set one more time to build the FP-tree.

After the tree is ready, the algorithm explores the tree in a bottom-up fashion. It looks for frequent itemsets ending in $e$ first, followed by $d, c, b$ and finally, $a$.

I will show the step-by-step process of finding frequent itemsets ending with $e$. The same process is then applied on $d, c, b$ and $a$.

(1) Gather all prefix paths - paths containing the node $e$.

(2) Calculate the support count for $e$ from the prefix paths. The required minimum support is 2, and {$e$} has support count 3. Thus, it is a frequent itemset.

(3) Convert the prefix paths into a conditional FP-tree using the following steps:

 - Update the support counts to include only the transactions that contain the item $e$.
 - Remove the nodes for $e$.
 - Eliminate items that have become infrequent.

(4) Use the conditional FP-tree for $e$ to solve the subproblems of finding frequent itemsets ending in $de, ce$ and $ae$.

  • Gather the prefix paths for $d$, and calculate the support count for {$d, e$}. Since it is equal to 2, {$d, e$} is a frequent itemset.

  • Construct the conditional FP-tree for $de$ (using step 3). The tree contains only one item ($a$) which is frequent. Thus, {$a, d, e$} is a frequent itemset.

  • Gather the prefix paths for $c$, and calculate the support count for {$c, e$}. Since it is equal to 2, {$c, e$} is a frequent itemset.

  • Construct the conditional FP-tree for $ce$ (using step 3). The tree contains only one item ($a$) which is infrequent. Thus, {$a, c, e$} is not a frequent itemset.

  • Gather the prefix paths for $a$, and calculate the support count for {$a, e$}. Since it is equal to 2, {$a, e$} is a frequent itemset.

The algorithm proceeds with the same devide-and-conquer approach to find all frequent itemsets ending with $d, c, b$ and $a$.
